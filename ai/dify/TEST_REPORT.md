# 一键游 Multi-Agent Dify 原型链路测试报告

测试日期：2026-07-15
测试应用：一键游 Multi-Agent 演示原型 v2
测试模式：Mock 数据，不调用真实天气、酒店、交通、门票或支付接口

## 测试结论

核心演示链路已跑通。8 条端到端用例全部到达预期业务分支，其中 3 类问题在测试过程中完成修复：明显意图误分类、DeepSeek 思考内容导致 JSON 解析失败、预订类型数组契约不一致。

## 本次修复

1. 在 `TravelState` 归一化节点增加高确定性意图兜底，修复“今天天气怎么样”被识别成 `general_qa` 的问题。
2. 对 DeepSeek 返回的 `<think>...</think>` 内容进行剥离，再提取 JSON，避免实体为空以及规划、评审、修改节点解析失败。
3. 将 `booking_type`、`requested_tools`、`selected_option_ids` 等字段统一归一化为数组，修复 `quote_types is not an array`。
4. 在候选、行程、软评审、二次校验和修改计划解析节点加入相同的 JSON 清洗规则。

## 剩余事项

- 当前所有外部工具都是 Mock，后续替换 FastAPI 工具时应保持相同输入输出契约。
- 完整规划链路在 DeepSeek v4-pro 上耗时较长，演示时建议预留 2 至 4 分钟。
- 交通缺参追问目前会询问目的地、日期和座位偏好，后续可改为只追问执行所必需的字段。
- Dify 工作区已达到 5/5 应用上限，因此本次直接更新了现有原型应用，没有新建副本。
- 应用保持未发布状态。
