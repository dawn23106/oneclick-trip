# 一键游 v3 链路测试用例

## 执行方式

先运行静态与状态测试：

```powershell
cd "C:\Users\20343\Documents\New project\oneclick-trip\ai\dify"
python build_v3_dsl.py
python validate_v3_dsl.py
```

然后在 Dify 导入 `oneclick-trip-multi-agent-v3.yml`，在同一对话中执行需要跨轮状态的用例。

## 核心用例

| ID | 输入/步骤 | 预期结果 |
| --- | --- | --- |
| V3-01 | `成都今天天气怎么样？` | 识别 `weather_query`，只选择天气工具，不进入偏好追问和完整行程规划；Flash 整理结果 |
| V3-02 | `成都有哪些特色美食？` | 进入普通查询/RAG 分支，只返回美食相关结果 |
| V3-03 | `从北京去成都玩3天，两个人，预算5000，喜欢美食，节奏轻松` | 进入两阶段研究；Pro 完成候选选择、规划和软评审；保存 `current_plan_json` |
| V3-04 | 完成 V3-03 后输入 `把第二天上午换成熊猫基地` | 从检查点加载上一版结构化方案，只修改目标时段，`plan_version` 增加 |
| V3-05 | 新对话直接输入 `确认预订` | 拒绝提交，返回 `NO_PENDING_DRAFT` |
| V3-06 | 输入 `预订上次方案里的酒店`，查看草稿后输入 `确认预订` | 只生成酒店草稿；第二轮校验同一会话、未过期、待确认后 Mock 提交 |
| V3-07 | 完成 V3-06 后再次输入 `确认预订` | 拒绝重复提交，返回 `DRAFT_NOT_CONFIRMABLE` |
| V3-08 | 在另一新对话中确认旧对话草稿 | 不得提交；草稿不会跨 `conversation_id` 读取 |
| V3-09 | `成都今天天气怎么样？模拟天气接口失败` | 首次工具调用失败，执行一次重试，返回 `MOCK_RETRY_SUCCESS`，不形成循环 |
| V3-10 | `成都今天天气怎么样？模拟不可恢复错误` | 天气有缓存能力，进入 `fallback`，必须说明数据时效 |
| V3-11 | `查北京到成都高铁，模拟不可恢复错误` | 无可用替代源时进入 `degraded`，明确失败且不伪造实时票价 |
| V3-12 | `帮我规划成都三日游，模拟酒店接口失败` | 阶段1酒店区域使用缓存降级，恢复轨迹传给候选选择节点 |
| V3-13 | `删除我喜欢徒步这个偏好` | 进入记忆管理分支，不触发旅行规划或实时工具 |

## 观察点

1. `intent_agent`、单项结果整理等简单节点显示 `deepseek-v4-flash`。
2. `planner`、`soft_reviewer`、`modify_agent` 等复杂节点显示 `deepseek-v4-pro`。
3. 单项天气问题不出现酒店、机票、门票工具调用。
4. 预订草稿提交后，Conversation Variable 中状态变为 `SUBMITTED`。
5. 任一失败分支最多出现一次重试，之后必须成功、降级或明确结束。
